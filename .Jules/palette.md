## 2023-10-25 - Terminal UI Progress Indicator for Headless Rust CLI
**Learning:** For a headless Rust CLI app, sending progress updates to `stderr` prevents polluting standard output if the user pipes it to a file. Furthermore, formatting these updates with ANSI escape codes (`\r\x1b[2K`) to overwrite the line and throttling the updates via a check against `std::time::Instant::now` provides a clean and responsive UX without stuttering. It's also crucial to conditionally enable this feature using `std::io::stderr().is_terminal()` so that progress messages aren't emitted in non-interactive environments (like CI).
**Action:** When adding progress feedback or UI elements to background/processing commands, write updates to `stderr`, utilize conditional checks for TTY (`is_terminal`), manage output updates smoothly via ANSI codes, and ensure the UI refresh is rate-limited.
